# SNL v2 로드맵

> 최종 목표: 로스트아크 캐릭터 조회 + 시뮬레이터 + 효율 분석 서비스
> 최종 수정: 2026-04-01

## 서비스 구성 (최종)

| 탭 | 설명 | 상태 |
|----|------|------|
| 능력치 | 장비, 보석, 특성, 각인, 아크그리드, 젬, 카드 | 장비 구현 완료, 나머지 미구현 |
| ~~아바타~~ | 제거 예정 | - |
| 스킬 | 스킬 트리 정보 (딜지분 계산은 추후) | 미구현 |
| 보유 캐릭터 | 같은 원정대 캐릭터 목록 | 미구현 |
| **시뮬레이터** | 원본 데이터 주입 → 장비/세팅 변경 → 상태 저장 | 미구현 |
| **효율 분석** | 시뮬레이터 상태 기반 LOPEC 동일 항목 계산 | 미구현 |

### 효율 분석 항목
- 치명타 적중률
- 치명타 피해량
- 공격 및 이동 속도
- 팔찌 효율
- 각인 총합 효율
- 메인노드 효율

## Step 1: 코드 최적화 + 아키텍처 정비

> 목적: 바이브코딩으로 작성된 코드를 실무 수준으로 정비하여 이후 작업의 기반을 만든다

### 1-1. 코드 컨벤션 통일
- [ ] ESLint 규칙 강화 (일관된 따옴표, 세미콜론 등)
- [ ] 전체 파일 lint + format 일괄 적용
- [ ] import 정리 (순서, 미사용 제거)

### 1-2. 컴파일 경고/에러 수정
- [ ] `npm run build` 전수 통과
- [ ] TypeScript strict 모드 경고 해결
- [ ] `eslint-disable` 주석 최소화 (any 타입 정리)

### 1-3. 모듈 분리
- [ ] `lib/` 재구성
  - `lib/api/` — API 호출 (lostark-api, korlark-api)
  - `lib/parser/` — 응답 파싱 (api-parser, tooltip-parser)
  - `lib/calc/` — 계산 로직 (효율, 시뮬레이션)
  - `lib/utils/` — 범용 유틸 (cn, extract-palette 등)

  - `components/layout/` — 레이아웃 (헤더, 검색바 등)
  - `components/character/` — 캐릭터 도메인 컴포넌트

### 1-4. 타입 정의 정비
- [ ] `types/character.ts` → 도메인별 분리 검토
  - `types/equipment.ts` — 장비/악세서리
  - `types/skill.ts` — 스킬
  - `types/profile.ts` — 프로필/스탯
  - `types/simulation.ts` — 시뮬레이터용 타입
- [ ] `any` 타입 제거 (특히 api-parser.ts)

## Step 2: 데이터 레이어 설계

> 목적: 시뮬레이터 → 효율 분석으로 이어지는 데이터 파이프라인의 기반

### 데이터 흐름
```
로스트아크 API → 파싱(api-parser) → CharData → 스토어 저장
                                        ↓
                              시뮬레이터(수정본 생성)
                                        ↓
                              효율 분석(계산 엔진)
```

### 2-1. 캐릭터 데이터 스토어
- [ ] 상태 관리 라이브러리 선택 (Context API vs Zustand)
- [ ] 스토어 구조 설계
  - `original: CharData` — API 원본 (읽기 전용)
  - `simulated: CharData` — 시뮬레이터 수정본
  - `isDirty: boolean` — 수정 여부
- [ ] Provider 구현 + 캐릭터 페이지에 연동

### 2-2. 영속화
- [ ] localStorage 저장/복원 전략
- [ ] 캐릭터별 키 관리
- [ ] TTL 기반 만료 처리
